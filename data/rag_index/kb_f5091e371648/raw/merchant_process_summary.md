# 商家关注的流程与环节（赛题核心总结）

> **赛题要求："并最终站在商家的角度，给出需要关注的流程和环节的总结。"**
> 本文档面向企业X电商运营 / 选品执行人员，**跨平台抽象出一份完整的"报名→上架→大促→售后" 全链路 SOP**。Agent 在 `checklist_builder` 节点应直接以本文档结构作为骨架。

---

## 1. 全链路阶段图

```
[T-45]            [T-30]            [T-15]          [T-7]           [T0]            [T+1]          [T+30]
  │                 │                 │               │               │              │              │
[资质年审]      [选品 + 价格盘]   [报名提报]      [预热]         [大促售卖]      [价保 + 售后] [复盘 + 结算]
```

---

## 2. 阶段 1：资质年审与赛事筹备（T-45 ～ T-30）

### 2.1 商家关注点

| 环节 | 检查项 | 风险等级 |
|------|--------|---------|
| 营业执照、商标证 | 在有效期；经营范围覆盖珠宝 | high |
| 贵金属生产 / 销售许可证 | 在有效期 | high |
| 国检证书 / CMA 实验室资质 | 不少于 6 份样品近 6 个月内的检测报告 | medium |
| 品牌天猫 / 京东 / 抖音店铺评分 | DSR ≥ 4.7（天猫），京东商智 ≥ 9.0 | high |
| 店铺违规记录 | 近 180 天无 A 级违规 | high |
| 财务体系 | 平台佣金、技术服务费已结清 | medium |
| 商品库存系统 | ERP / OMS / WMS 与平台库存接口正常 | medium |

### 2.2 Agent 在该阶段应输出

- **资质过期清单**：从 `../chow_tai_fook/company_qualifications.md` 中拉取所有资质证书，按过期日排序。
- **违规风险提示**：店铺评分接近阈值（如 DSR = 4.71 离 4.7 仅 0.01 差距）时主动预警。

---

## 3. 阶段 2：选品 + 价格盘整（T-30 ～ T-15）

### 3.1 选品执行步骤

1. **拉取主推 SKU 池**
   - 维度：黄金 / 镶嵌 / 一口价 / 古法金 / 5G 黄金 / 钻石 / 翡翠 / 珍珠（参见 `../chow_tai_fook/sku_catalog.md`）。
   - 数据筛选：近 90 天店内销量 ≥ 50 件、好评率 ≥ 95%、退货率 ≤ 5%。
2. **匹配活动门槛**
   - 数量门槛：天猫 200+，京东 150+，淘宝百亿补贴 ≥ 1。
   - 新品门槛：近 90 天上新占比 ≥ 30%（天猫）。
   - 库存门槛：单 SKU 库存承诺 ≥ 500 件 或 日均销量 × 10 天。
3. **价格盘整**
   - 拉取 SKU 近 14 天 / 近 30 天 / 近 90 天 / 近 365 天最低成交价。
   - 全网比价：京东、抖音、品牌官网、得物、拼多多同款 SKU 实时价。
   - 计算让利幅度：(划线价 - 活动价) / 划线价 ≥ 15%（基本门槛）。
4. **互斥活动盘点**
   - 同 SKU 是否在其他平台同期报名？
   - 同店铺是否同期跑超品日 + 聚划算？
   - 是否触发"品牌日 vs 品类日"互斥？

### 3.2 Agent 在该阶段应输出

- **价格穿底预警**：标记不满足"近 30 天最低 / 全网最低"的 SKU。
- **互斥冲突清单**：明确列出与当前规则互斥的其他活动名称。
- **库存不足预警**：单 SKU 实际库存 < 承诺值时主动提示。

### 3.3 易混淆维度提醒

> **黄金计重款** vs **一口价金饰**：
> - 计重款：克数 × 实时金价 + 工费，价格随金价波动；不进入"超品日"价格管控（因为单价不固定）。
> - 一口价款：固定价格，参与价格管控。
> 选品时务必区分，避免把计重款误填入"活动价 ≤ 近 30 天最低价" 的池。

## 4. 阶段 3：报名提报（T-15 ～ T-7）

### 4.1 报名材料清单

| 材料 | 来源 | 备注 |
|------|------|------|
| SKU 列表（Excel / CSV） | ERP 导出 | 含 SKU 编码、品名、克重、4C、活动价 |
| 价格清单 | 价格盘整结果 | 划线价、活动价、近 X 天最低价、全网最低价对比 |
| 库存承诺 | WMS | 单 SKU ≥ 500 件 或 等价金额 |
| 商品证书电子件 | 国检 / GIA / NGTC | 单独打包，每个 SKU 对应一份 |
| 营业执照、商标证 | 法务 | 加盖公章扫描件 |
| 活动履约协议 | 商务 | 双方签字盖章 |

### 4.2 平台小二审核要点

| 小二关注 | 商家应对 |
|----------|---------|
| 活动价是否 ≤ 近 X 天最低 | 提供订单库截图佐证 |
| 让利幅度是否 ≥ 15% | 提供让利计算明细 |
| 库存承诺是否真实 | 提供 WMS 实时库存截图 |
| 新品占比 | 提供近 90 天上新 SKU 列表 |
| 全网价是否最低 | 提供 5 大平台比价表 |

### 4.3 Agent 在该阶段应输出

- **报名材料缺失清单**：自动核对必交材料。
- **审核被驳回原因建模**：解析平台审核反馈，定位是价格、库存、资质中的哪一项。

---

## 5. 阶段 4：预热（T-7 ～ T-1）

### 5.1 预热动作清单

1. **店铺装修**：首页 / 商详页打上活动 banner、利益点角标。
2. **券与红包配置**：店铺券、品牌满减、定金预售在后台配置完成。
3. **私域触达**：会员短信、企业微信、淘宝订阅消息、京东 PLUS 触达。
4. **内容投放**：抖音 / 小红书种草内容，直播预告。
5. **库存调拨**：将目标 SKU 库存预留 80% 给当日，剩余 20% 留售后。
6. **客服培训**：统一活动话术、价保规则、跨店满减计算方法。

### 5.2 Agent 在该阶段应输出

- **触达任务清单**：自动生成"会员短信 / 企业微信 / 直播预热" 的 to-do。
- **客服话术包**：根据规则原文自动生成"价保 / 退差 / 跨店满减" 的标准回复。

## 6. 阶段 5：大促售卖（T0）

### 6.1 当日盯盘清单（每小时检查）

| 项 | 检查动作 |
|----|---------|
| 价格 | 价格、券、跨店满减是否正常生效 |
| 流量 | 品牌词 / 品类词排名是否稳定 |
| 库存 | 主推 SKU 是否快速售罄，触发缺货告警 |
| 转化 | 加购 / 下单转化率是否符合预期 |
| 客诉 | 价保、券失效、宣传不实类投诉是否集中 |
| 物流 | 仓库出库速率、订单堆积情况 |

### 6.2 异常处理 SOP

| 异常 | 处理动作 | 上报对象 |
|------|---------|---------|
| 主推 SKU 价格穿底 | 30 分钟内调价或下架 | 商务 + 法务 |
| 流量掉位 | 联系平台小二 | 运营 |
| 库存告急 | 紧急调拨 / 限购 | 仓配 |
| 客诉集中 | 客服话术升级 | 客服主管 |
| 物流堆积 | 启用备用仓 | 仓配 |

### 6.3 Agent 在该阶段应输出

- **实时盯盘看板**：自动汇总价格、流量、库存、转化、客诉。
- **异常告警**：触发阈值时自动推送处理建议。

## 7. 阶段 6：售后（T+1 ～ T+30）

### 7.1 售后动作清单

| 环节 | 动作 | 时间窗口 |
|------|------|---------|
| 价保退差 | 每日跑批退差 | T+1 ～ T+15（天猫）/ T+30（京东）/ T+45（京东 PLUS） |
| 7 天无理由退货 | 退货审核、换货发货 | T+1 ～ T+7 |
| 30 天质量问题 | 检测、维修、换货 | T+1 ～ T+30 |
| 假一赔十 / 假一赔三 | 鉴定、赔付 | 视情况而定 |
| GMV 结算 | 与平台对账 | T+30 |
| 保证金解冻 | 平台返还 | T+30（天猫）/ T+90（百亿补贴） |
| 数据复盘 | 导出看板、分析 | T+7 |

### 7.2 Agent 在该阶段应输出

- **价保退差任务流**：自动生成每日退差清单 + 异常账单。
- **复盘报告骨架**：自动生成 GMV、新客、复购、跨品类拉新等核心指标的报告模板。

## 9. 总结：商家最容易漏掉的 10 个环节

1. **历史最低价回溯**：尤其是近 365 天某次秒杀价被忘记。
2. **全网比价**：京东、抖音、品牌官网的价格未拉平。
3. **库存承诺与实际库存差距**：报名时承诺 1000 件，实际 200 件。
4. **互斥活动重叠**：同 SKU 既报超品日又报百补。
5. **新品占比不足**：近 90 天上新 SKU 数量不够。
6. **划线价虚高**：使用厂商指导价 / 吊牌价。
7. **赠品估值超 30%**：变相打折触发价格管控。
8. **价保规则不一致**：天猫 15 天 vs 京东 30/45 天，客服误答。
9. **跨店满减叠加错配**：百亿补贴 SKU 误挂跨店满减标签。
10. **资质过期**：营业执照、贵金属许可证、商标证忘记年审。

> Agent 在 `checklist_builder` 节点应针对每一个环节生成显式检查项，标记为 high 风险，并在 `risk_points` 中加粗提示。
