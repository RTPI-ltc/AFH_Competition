# 双 11 大促规则知识库

> **聚焦赛题方向 C**：双 11 是周大福电商最大节点。与日常品牌日的关键差异：**多波次时间结构 + 定金预售 + 跨期价保 30 天 + 近 30/365 天双重价格校验**。

---

## 一、双 11 时间结构（影响 Agent 解析"活动期间"语义）

| 阶段 | 时间（典型） | Agent 关注 |
|------|-------------|-----------|
| 蓄水期 | 10/20 起 | 私域预热、加购、领券 |
| 预售期（定金） | 10/24 20:00 ～ 10/31 | 付定金，定金膨胀 2–5 倍 |
| 第一波 | 10/31 20:00 ～ 11/03 | 付尾款 + 现货售卖 |
| 余热期 | 11/04 ～ 11/10 | 持续售卖 |
| 高潮期 | 11/10 20:00 ～ 11/11 24:00 | 全场大促 + 跨店满减升级 |
| 返场期 | 11/12 ～ 11/14 | 库存清理 |

> Agent 用法：解析"活动期间不得调价" 这类规则时，需识别活动覆盖了上述哪几个阶段；定金预售阶段单独定价。

---

## 二、价格管控（最严，超 618 一档）

| 检查项 | 规则 |
|--------|------|
| 活动价 ≤ 近 30 天最低价 | 标准滚动 30 天 |
| 活动价 ≤ 历史 365 天最低 × 95% | **比 618 加严**：相对去年同期再降 5% |
| 划线价合规 | 近 30 天有实际成交、未划线 |
| 定金 + 尾款总价 ≤ 现货价 | 预售总价不可超过现货活动价 |
| 价保期 | **30 天跨期价保**（覆盖整个双 11 周期） |
| 跨店满减 | 升级到"每满 300 减 50 + 每满 1000 减 200" 双档位 |

---

## 三、SKU 选品条件

| 检查项 | 门槛 |
|--------|------|
| 报名 SKU 数 | ≥ 300 款（比超品日多） |
| 超级单品数 | ≥ 8 款 |
| 单 SKU 近 90 天销量 | ≥ 100 件（高于超品日的 50 件） |
| 单 SKU 库存承诺 | ≥ 1000 件 或 日均销量 × 60 天 |
| 新品占比 | ≥ 40%（高于超品日 30%） |
| 直播专场 SKU | ≥ 30 款 |

---

## 四、双 11 特有的互斥规则

| 互斥项 | 说明 |
|--------|------|
| 与品牌日 | 同年内已报本店品牌日的 SKU，双 11 价格须再降 5%（不是禁止，是加严） |
| 与品类日 | 同期不可重复报名 |
| 与百亿补贴 | 双 11 期间百亿补贴 SKU 单独运营，不进双 11 主会场 |
| 与聚划算 | 双 11 高潮期聚划算 SKU 全部纳入主会场，不需单独报名 |
| 与定金预售 vs 现货 | 同 SKU 不可同期既走预售又走现货（除非平台特别许可） |

## 五、Agent 解析双 11 规则的特殊注意

### 5.1 多波次价格切换

```text
[规则] "活动期间不可调价"
[歧义] 活动期间是指 10/24-11/14 全程，还是仅 11/10-11/11 高潮期？
[澄清问句] "活动期间是涵盖预售期、第一波、余热期、高潮期、返场期，还是仅指其中某几个阶段？"
[Agent 默认] 全程，confidence 0.7
```

### 5.2 定金预售价格陷阱

```text
[规则] "定金膨胀 2 倍"
[Agent 解析]
- 付定金阶段：消费者付定金 D 元，膨胀后视为已付 2D 元抵尾款
- 尾款阶段：尾款 = 总价 - 2D
- 关键校验：(D + 尾款) 是否 ≤ 现货活动价
- 风险：若膨胀过大导致总价 < 近 30 天最低价 → 穿底
```

### 5.3 跨期价保

```text
[规则] "全周期 30 天价保"
[Agent 解析]
- 价保口径：以"到手价"计算
- 时段覆盖：10/24 付定到 12/14 全程
- 商家执行：每日跑批退差，长达 50 天的价保窗口
- 风险：长价保导致退差账目复杂，需 OMS 自动化
```

## 六、常见踩坑

| 风险点 | 触发场景 | 风险等级 |
|--------|---------|---------|
| 365 天历史最低 × 95% 穿底 | 用今年 618 的活动价填，没再降 5% | high |
| 定金 + 尾款 > 现货价 | 膨胀比例错配 | high |
| 互斥与品牌日"加严降价 5%"被误读为"禁止" | 规则理解错误 | medium |
| 直播专场库存 < 30 款门槛 | 直播 SKU 数量不足 | medium |
| 跨期价保账目混乱 | 没有自动化退差 | high |
