# 天猫 618 大促规则知识库（赛题示例对应）

> **本文档对标赛题方向 C 在 manual.pdf 中给出的标准示例规则**：
>
> > 天猫 618 大促选品规则：
> > 1. 参与商品必须满足：近 30 天销量 ≥ 100 件；好评率 ≥ 95%；库存 ≥ 500 件。
> > 2. 价格要求：活动价不得高于近 30 天最低价；折扣力度 ≥ 7 折。
> > 3. 品类限制：黄金类目单店最多 5 个 SKU；钻石类目单店最多 10 个 SKU。
> > 4. 互斥规则：已参加"品牌日"活动的商品不可重复报名。
>
> 本文档对该示例规则做完整领域化展开，作为 Agent 端到端 demo 时的对标规则。

## 一、赛题示例规则的解析样板

### 1.1 拆解到检查项（Agent 输出格式）

| 序号 | 检查项 | 数据来源 | 期望值 | 置信度 | 风险等级 |
|------|--------|---------|--------|--------|---------|
| C1 | 近 30 天销量 | `sku.last_90d_sales` 推算或直接字段 | ≥ 100 件 | 0.95 | high（数值型） |
| C2 | 近 30 天好评率 | `sku.review_rate` | ≥ 95% | 0.95 | high（数值型） |
| C3 | 库存 | `sku.stock` | ≥ 500 件 | 0.95 | high（数值型） |
| C4 | 活动价 ≤ 近 30 天最低价 | `sku.list_price_rmb` vs `sku.last_30d_min_price` | 不大于 | 0.95 | high（价格类） |
| C5 | 折扣力度 | (划线价 - 活动价) / 划线价 | ≥ 30% | 0.95 | high（数值型） |
| C6 | 黄金类目 SKU 数 | `sku.category_l1 == '黄金'` 计数 | ≤ 5 | 0.95 | high（数量型） |
| C7 | 钻石类目 SKU 数 | `sku.category_l1 == '镶嵌'` 且含钻 计数 | ≤ 10 | 0.95 | high（数量型） |
| C8 | 互斥：已参加品牌日 | `sku.active_campaigns` 是否包含品牌日 | 不包含 | 0.9 | high（互斥型） |

### 1.2 决策流程（decision_flow，赛题要求）

```
Step 1: 商品近 30 天销量 ≥ 100 件？
  → 是：继续 / 否：不符合，排除

Step 2: 好评率 ≥ 95%？
  → 是：继续 / 否：不符合，排除

Step 3: 库存 ≥ 500 件？
  → 是：继续
  → 否：检查补货周期 lead_time_days 是否 ≤ 距活动天数
    → 是：medium，建议立即补货
    → 否：high，建议下调承诺或下架

Step 4: 活动价 ≤ 近 30 天最低价？
  → 是：继续 / 否：high，建议调价或退出

Step 5: 折扣力度 ≥ 30%？
  → 是：继续 / 否：medium，建议下调活动价

Step 6: 该 SKU 已参加品牌日？
  → 是：high，互斥，不可报名
  → 否：继续

Step 7: 该 SKU 类目计数是否超限？
  → 黄金类：> 5 → high，需取舍
  → 钻石类：> 10 → high，需取舍
  → 否：通过
```

### 1.3 边界反例（counter_examples，赛题要求 2-3 个）

| 反例 | 触发的规则 | 用途 |
|------|----------|------|
| SKU 近 30 天销量 99 件 | C1（恰好不满足销量 ≥ 100） | 验证 Agent 是否正确处理"恰好 < 阈值"的边界 |
| SKU 活动价 = 近 30 天最低价 - 1 元 | C4（恰好满足） | 验证 Agent 是否处理"刚好通过"边界 |
| SKU 折扣力度 = 7 折，库存 = 500，但已参加品牌日 | C5 + C8 | 验证 Agent 在多规则共同作用下能否正确得出"不通过"（互斥优先） |
| SKU 折扣力度 = 30.01%（恰好 7 折零点几） | C5 边界 | 验证 Agent 浮点精度处理 |

### 1.4 需要主动澄清的歧义点（clarification_questions）

| # | 歧义点 | 标准澄清问句 |
|---|-------|--------------|
| Q1 | "近 30 天"是滚动 30 天还是自然月？ | "本规则中'近 30 天'是按活动开始日往前推 30 个自然日（滚动），还是上一个完整自然月？" |
| Q2 | "好评率 ≥ 95%" 是近 30 天还是历史累计？ | "好评率统计时窗是近 30 天、近 90 天还是全部历史？" |
| Q3 | "库存 ≥ 500 件" 是报名时还是活动开始时？ | "库存校验时点是报名提报时、平台审核时，还是活动开始日 0:00？" |
| Q4 | "活动价"是否含跨店满减？ | "活动价是指商品标价，还是含跨店满减后的到手价？" |
| Q5 | "折扣力度 ≥ 7 折"的分母是哪一个价？ | "折扣 = 1 - 活动价 / 划线价，还是 1 - 到手价 / 划线价，还是 1 - 活动价 / 吊牌价？" |
| Q6 | "品牌日"指本店品牌日还是同行业任何品牌日？ | "互斥规则中的品牌日是指本店在本平台的品牌日，还是同集团其他店铺 / 其他平台的品牌日？" |
| Q7 | "黄金类目"和"钻石类目"是按类目编码还是商品标签？ | "类目限制是按平台后台一级类目，还是商品所打的'黄金'/'钻石'标签？" |
| Q8 | 黄金计重款是否纳入"折扣力度 ≥ 7 折"？ | "黄金计重款价格随金价波动，是否纳入折扣力度校验？" |

> Agent 在 `rule_parser` 节点应主动生成 Q1–Q5 的澄清问题，因为这些是**所有平台都会出现的口径歧义**。Q6–Q8 是珠宝品类专属歧义。

## 二、618 大促与日常超品日的关键差异

| 维度 | 日常超品日 | 618 大促 |
|------|-----------|---------|
| 时间结构 | 单日 | **多波次**（第一波 5/31-6/3 + 高潮期 6/16-6/18 + 返场 6/19-6/20） |
| 跨店满减 | 每满 300 减 50 | 每满 300 减 50 **升级到每满 1000 减 200** |
| 价保 | 15 天 | **跨期价保 30 天**（覆盖整个 618 周期） |
| 价格回溯 | 近 30 天 | **近 30 天 + 近 365 天双重校验**（要求活动价 ≤ 历史最低 × 95%） |
| 定金预售 | 通常无 | **有定金预售**（5/24-5/27 付定金，5/31-6/3 付尾款），定金膨胀 2–5 倍 |
| 新品要求 | ≥ 30% | ≥ 30%（不变） |
| 直播 | 店播日均 8h | 店播日均 16h + 总裁直播专场 |
