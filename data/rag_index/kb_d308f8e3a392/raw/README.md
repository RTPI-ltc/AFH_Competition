# 企业X内部知识库（Chow Tai Fook Internal KB）

> 本知识库是"执行辅助 Agent"理解企业X企业语境的主要上下文。
> **注意**：本目录中的资质、SKU、供应链、组织架构等数据均为黑客松演示用的**模拟样例**，结构与字段尽可能贴近真实场景，便于 Agent 端到端跑通；正式落地时需要替换为企业X内部权威系统（ERP / OMS / WMS / 中台）抓取的实时数据。

## 文件索引

| 文件 | 内容 | Agent 调用场景 |
|------|------|----------------|
| `company_qualifications.md` | 企业资质、品牌主体、法律牌照、平台店铺账号 | 平台报名前的资质核验 |
| `product_certifications.md` | 商品类目对应的国检 / GIA / NGTC 等证书要求 | 选品时确认 SKU 是否具备活动门槛要求的证书 |
| `sku_catalog.md` | SKU 数据库样例（含 30 条多品类样例 SKU + 在售/历史活动字段） | 选品入池、价格盘整、库存承诺、互斥检测 |
| `supply_chain.md` | 工厂代码、补货周期、大促备货策略（已精简，只留 C 方向需要的部分） | 库存承诺核验、补货预案 |
| `human_review_routing.md` | 不同规则类型 → 人工接管角色的路由表 | `human_review_gate` 节点输出"需人工确认"时的路由 |

## 数据使用原则（与赛题对齐）

1. **以"内部权威数据" 为最高优先级**：当活动规则模糊时，Agent 优先用内部知识库回答"我们是否满足"，而不是依赖外部猜测。
2. **数据缺失自动标注 `requires_human=true`**：例如 SKU 缺少国检证书电子件、库存承诺与 WMS 不匹配时，Agent 不擅自判定通过，而是输出"建议人工确认"。
3. **数据敏感度**：本目录中的所有数值（如 SKU 单价、库存量、利润率）均为**模拟值**。生产环境必须通过加密通道访问 ERP / OMS / WMS。

## 与其他知识库的协作

- 平台知识库（`../platform/`）告诉 Agent **平台门槛**；
- 本知识库告诉 Agent **企业当前状态**；
- 行业知识库（`../jewelry_industry/`）告诉 Agent **品类语义**。

三者在 `rule_parser` 与 `checklist_builder` 节点中同时注入 prompt context。
