# 人工接管路由（Human Review Routing）

> **聚焦赛题方向 C**：Agent 在 `human_review_gate` 节点输出"需人工确认"时，根据规则类型路由到对应角色。本文档只保留路由表，不包含组织架构图、跨角色协作流程（属于赛题方向 B 协作沟通）。

---

## 路由规则表

| 规则类型 | 触发场景示例 | 路由角色 | SLA |
|---------|--------------|---------|-----|
| **价格规则歧义** | "近 30 天最低价" 口径未明确；让利幅度计算口径有争议 | 价格管理 + 品类经理 | 2 小时内反馈 |
| **资质 / 证书有效性** | 商标证临期 6 个月内；证书 PDF 文件丢失 | 法务 | 4 小时内反馈 |
| **库存承诺** | 实际库存 < 承诺；补货周期紧张 | 供应链 + 品类经理 | 2 小时内反馈 |
| **互斥活动判定** | 同 SKU 已在其他平台同期报名；品牌日 vs 品类日冲突 | 平台运营 | 1 小时内反馈 |
| **新品 / 选品口径** | "30% 新品" 是按 SKU 数还是 GMV 占比 | 品类经理 | 4 小时内反馈 |
| **让利预算上限** | 单 SKU 让利 ≥ 20% 或整场让利 ≥ 500 万 | 电商事业部总监 | 1 工作日 |
| **价格穿底应急** | 当日发现外渠穿底，需调价或下架 | 价格管理 + 平台运营 + 法务 | 30 分钟内 |
| **平台违规申诉** | 收到平台违规处罚 | 法务 + 平台运营 | 4 小时内 |

---

## Agent 在 UI 上的展示规范

当 Agent 输出 `requires_human=true` 时，前端卡片应同时展示：

1. **风险点描述** + **风险等级**（high / medium / low）
2. **建议路由角色**（按上表）
3. **建议 SLA**
4. **澄清问题列表**（如果是规则歧义类）
5. **当前 Agent 的初步判定建议**（供人工 review 参考，不替代决策）

```
┌──────────────────────────────────────────────┐
│ ⚠ 风险点：活动价 198 元 < 近 30 天最低价 188 元 │
│ 等级：high                                     │
│ 建议路由：价格管理 + 品类经理                    │
│ SLA：2 小时内反馈                              │
│ Agent 初判：建议调价至 ≥ 188 元 或退出报名      │
└──────────────────────────────────────────────┘
```
