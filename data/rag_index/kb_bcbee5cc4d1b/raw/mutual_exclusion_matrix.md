## 2. 跨平台的互斥关系（重点）

> 跨平台同 SKU 报名通常**不互斥**，但价格穿底约束极易出问题。Agent 主要做**穿底风险检测**。

| 同 SKU 跨平台报名场景 | 是否允许 | 风险 |
|----------------------|---------|------|
| 天猫超品日 + 京东超品日 | ✅ 允许 | 价格须保持全网一致 |
| 天猫超品日 + 淘宝百亿补贴 | ❌ 互斥 | 阿里集团内部互斥 |
| 京东百亿补贴 + 淘宝百亿补贴 | ✅ 允许 | 须保持全网最低承诺 |
| 抖音直播间 + 天猫商详 | ✅ 允许 | 抖音直播价不可低于天猫近 30 天最低 |
| 任何电商平台 + 内部清仓季 | ❌ 强烈不建议 | 内部清仓价穿底电商管控 |

## 4. Agent 互斥检测算法

### 4.1 单 SKU 互斥检测

```python
def detect_mutual_exclusion(sku, target_campaign):
    """
    sku.active_campaigns: list of campaign IDs the SKU is currently in
    target_campaign: campaign ID to register

    Returns: (is_exclusive, conflict_campaign, severity)
    """
    matrix = load_exclusion_matrix()
    for current in sku.active_campaigns:
        rule = matrix.lookup(current, target_campaign)
        if rule == "exclusive":
            return True, current, "high"
        if rule == "exclusive_with_relaxation":
            return True, current, "medium"  # 大促期豁免、可申请豁免
    return False, None, None
```

### 4.2 整批 SKU 互斥扫描（报名前）

```python
def scan_batch_for_exclusion(sku_list, target_campaign):
    conflicts = []
    for sku in sku_list:
        ex, conflict_camp, sev = detect_mutual_exclusion(sku, target_campaign)
        if ex:
            conflicts.append({
                "sku_id": sku.sku_id,
                "conflict_campaign": conflict_camp,
                "severity": sev,
                "suggestion": "二选一：退出 " + conflict_camp + " 或不报 " + target_campaign
            })
    return conflicts
```

## 5. 互斥规则的歧义识别（用于 rule_parser）

| 规则原文 | 歧义点 | 标准澄清 |
|----------|-------|----------|
| "已参加品牌日不可重复报名" | "品牌日"是同平台、同集团跨平台、还是历史所有期次？ | 默认按"同平台同店本期次" |
| "互斥活动" | 具体哪些算互斥？ | 引用本矩阵列表 |
| "二选一" | 二选一的对象是 SKU 还是店铺？ | 默认 SKU 级 |
| "全店参加" | 是单一店铺还是集团下所有店铺？ | 默认单一店铺 |

## 6. 矩阵维护说明

- 本矩阵每个大促前需复核（平台规则会调整）。
- 实际落地时，Agent 应通过 API 拉取平台最新互斥说明覆盖本矩阵。
- 黑客松版本：本矩阵静态写死，覆盖典型场景；产品化版本接入平台规则中心。
