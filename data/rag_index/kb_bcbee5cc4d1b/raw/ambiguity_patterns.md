# 规则歧义模式库（Ambiguity Patterns）

> Agent 在 `rule_parser` 节点解析活动规则时，应对照本文档识别歧义点，并主动生成标准澄清问句。
> 每一类歧义都包含：**触发关键词 → 可能解读 → 默认假设 → 标准澄清问句**。

---

## 1. 时间窗口类歧义

### 1.1 "近 X 天" 是滚动还是自然月

| 项 | 内容 |
|----|------|
| 触发关键词 | "近 30 天"、"近 7 天"、"近 14 天"、"近 90 天"、"近 365 天"、"过去 30 天" |
| 可能解读 | A) 滚动 X 天（活动开始日往前推 X 个自然日） / B) 上一个完整自然月 |
| 默认假设 | 按**滚动 X 天**处理 |
| 标准澄清问句 | "本规则中'近 X 天'是按活动开始日往前推 X 个自然日（滚动窗口），还是上一个完整自然月？" |
| 平台默认 | 天猫 30 天滚动 / 京东 14 天滚动 / 百补 90 天滚动 |

### 1.2 "活动期间" 覆盖哪个时段

| 项 | 内容 |
|----|------|
| 触发关键词 | "活动期间"、"大促期间"、"双 11 全程"、"618 期间" |
| 可能解读 | A) 整个大促周期（含预热、预售、第一波、高潮、返场） / B) 仅高潮日 |
| 默认假设 | **全程**（更严格） |
| 标准澄清问句 | "本规则中'活动期间'是涵盖预热、预售、第一波、高潮期、返场期全程，还是仅指其中某几个阶段？" |

### 1.3 "活动开始时" vs "报名时" vs "审核时"

| 项 | 内容 |
|----|------|
| 触发关键词 | "库存 ≥ X 件"、"销量 ≥ X 件"、"好评率 ≥ X%" |
| 可能解读 | A) 报名提报时 / B) 平台审核时 / C) 活动开始日 0:00 |
| 默认假设 | **活动开始日 0:00**（最严） |
| 标准澄清问句 | "本规则中库存/销量/好评率的校验时点是报名提报时、平台审核时，还是活动开始日 0:00？" |

## 2. 价格类歧义

### 2.1 "活动价" vs "到手价" vs "划线价"

| 项 | 内容 |
|----|------|
| 触发关键词 | "活动价 ≤ X"、"到手价 ≤ X"、"原价"、"划线价" |
| 可能解读 | A) 商家标价（不含优惠） / B) 消费者实付（含跨店满减、券、红包） |
| 默认假设 | "活动价" = 商家标价；"到手价" = 实付 |
| 标准澄清问句 | "本规则中'活动价'是指商家标价（不含跨店满减），还是含优惠后的到手价？" |

### 2.2 "全网最低" 的范围

| 项 | 内容 |
|----|------|
| 触发关键词 | "全网最低"、"全平台最低"、"最低价" |
| 可能解读 | A) 全部电商平台（天猫+京东+抖音+拼多多+得物+品牌官网） / B) 仅本平台 / C) 仅本平台 + 品牌官网 |
| 默认假设 | **全部电商平台**（最严） |
| 标准澄清问句 | "本规则中'全网最低'是指全部电商平台（含京东、抖音、拼多多、得物、品牌官网等），还是仅本平台？" |

### 2.3 "折扣力度 ≥ X 折" 的分母

| 项 | 内容 |
|----|------|
| 触发关键词 | "折扣 ≥ X 折"、"让利 ≥ X%"、"X 折起" |
| 可能解读 | A) 划线价 / B) 吊牌价 / C) 厂商指导价 |
| 默认假设 | **划线价** |
| 标准澄清问句 | "本规则中折扣力度的分母是商详页划线价、吊牌价，还是厂商指导价？" |

### 2.4 "让利 ≥ X%" 的让利构成

| 项 | 内容 |
|----|------|
| 触发关键词 | "商家让利 ≥ X%"、"让利幅度" |
| 可能解读 | A) 仅商家自身让利 / B) 商家 + 平台补贴 / C) 商家 + 平台补贴 + 主播让利 |
| 默认假设 | **商家 + 平台补贴**（百亿补贴口径） |
| 标准澄清问句 | "本规则中'让利幅度'包含哪些构成（商家自身、平台补贴、主播让利）？由谁出资分担多少？" |

### 2.5 "近 30 天最低价" 是否含历史促销价

| 项 | 内容 |
|----|------|
| 触发关键词 | "近 30 天最低成交价" |
| 可能解读 | A) 包含历史秒杀 / 闪购等极端低价 / B) 不含秒杀类一次性低价 |
| 默认假设 | **包含**（按订单库实际成交） |
| 标准澄清问句 | "近 30 天最低价是否包含历史秒杀、闪购等极端低价？" |

## 3. 品类 / 选品类歧义

### 3.1 "类目" 是平台编码还是商品标签

| 项 | 内容 |
|----|------|
| 触发关键词 | "黄金类目"、"钻石类目"、"金饰类目" |
| 可能解读 | A) 平台后台一级 / 二级类目编码 / B) 商品自打的标签 |
| 默认假设 | **平台后台类目编码** |
| 标准澄清问句 | "本规则中'类目'是按平台后台一级/二级类目编码，还是商品自打的'黄金'/'钻石'等标签？" |

### 3.2 "新品" 的定义

| 项 | 内容 |
|----|------|
| 触发关键词 | "新品占比 ≥ X%"、"上新 SKU" |
| 可能解读 | A) 近 90 天上架 / B) 近 30 天上架 / C) 首次上线 |
| 默认假设 | **近 90 天首次上架** |
| 标准澄清问句 | "本规则中'新品'指近 90 天首次上架、近 30 天首次上架，还是其他时窗？" |

### 3.3 黄金计重款是否纳入折扣校验

| 项 | 内容 |
|----|------|
| 触发关键词 | "折扣 ≥ X 折"（同时品类含黄金计重款） |
| 可能解读 | A) 仅一口价款纳入折扣校验 / B) 计重款也纳入（按当日金价为基准） |
| 默认假设 | **仅一口价款纳入**（计重款价格随金价波动，常被豁免） |
| 标准澄清问句 | "黄金计重款（价格随金价波动）是否纳入'折扣 ≥ X 折'校验？" |

### 3.4 "求婚钻戒" 的范围

| 项 | 内容 |
|----|------|
| 触发关键词 | "求婚钻戒"、"婚戒" |
| 可能解读 | A) 仅主石 ≥ 30 分单钻戒 / B) 包括对戒、群镶戒指 |
| 默认假设 | **仅主石 ≥ 30 分单钻戒**（行业惯例） |
| 标准澄清问句 | "'求婚钻戒'是仅指主石 ≥ 30 分的单钻戒，还是包含对戒、群镶戒指？" |

## 4. 互斥类歧义

### 4.1 "已参加 X 活动不可重复报名"

| 项 | 内容 |
|----|------|
| 触发关键词 | "已参加品牌日不可再报"、"互斥"、"二选一" |
| 可能解读 | A) 同平台同店本期次 / B) 同平台同店历史所有期次 / C) 同集团跨平台 |
| 默认假设 | **同平台同店本期次（同年内）** |
| 标准澄清问句 | "互斥规则中的'已参加'是指同店同期、同店历史，还是同集团跨平台的所有期次？" |

### 4.2 "全店参加" vs "全品牌参加"

| 项 | 内容 |
|----|------|
| 触发关键词 | "本店所有 SKU 必须参加"、"品牌全店参加" |
| 可能解读 | A) 单一店铺所有 SKU / B) 同集团所有店铺所有 SKU |
| 默认假设 | **单一店铺所有 SKU** |
| 标准澄清问句 | "本规则中'全店参加'是指单一店铺所有 SKU，还是同集团所有店铺所有 SKU？" |

## 5. 数量 / 单位类歧义

### 5.1 "SKU 数 ≤ X" 的统计口径

| 项 | 内容 |
|----|------|
| 触发关键词 | "黄金类 SKU ≤ 5"、"单店最多 X 个 SKU" |
| 可能解读 | A) 报名 SKU 数 / B) 在售 SKU 数 / C) 平台后台 SPU 数 |
| 默认假设 | **报名 SKU 数** |
| 标准澄清问句 | "本规则中 SKU 计数是按报名 SKU 数、在售 SKU 数，还是平台后台 SPU 数？" |

### 5.2 "好评率 ≥ X%" 的统计时窗

| 项 | 内容 |
|----|------|
| 触发关键词 | "好评率 ≥ 95%"、"差评率 ≤ X%" |
| 可能解读 | A) 近 30 天 / B) 近 90 天 / C) 全部历史 |
| 默认假设 | **近 30 天** |
| 标准澄清问句 | "好评率统计时窗是近 30 天、近 90 天还是全部历史？" |

## 6. 主体类歧义

### 6.1 "本店" vs "集团"

| 项 | 内容 |
|----|------|
| 触发关键词 | "本店"、"店铺"、"商家" |
| 可能解读 | A) 单一店铺账号 / B) 同集团所有店铺 |
| 默认假设 | **单一店铺账号** |
| 标准澄清问句 | "本规则中的'本店'是指单一店铺账号（如周大福官方旗舰店），还是同集团所有店铺（含 LOLA、SOINLOVE、T MARK 等）？" |

### 6.2 "品牌" 含义

| 项 | 内容 |
|----|------|
| 触发关键词 | "品牌让利 ≥ X%"、"品牌让利预算" |
| 可能解读 | A) 主品牌 / B) 集团下所有子品牌 |
| 默认假设 | **主品牌** |
| 标准澄清问句 | "本规则中的'品牌'是指主品牌还是集团下所有子品牌？" |

## 7. 歧义检测的标准输出格式

Agent 在 `rule_parser` 输出 `parsed_rules` 时，每一条规则若识别到歧义应附加以下字段：

```json
{
  "id": "R1.1",
  "rule": "近 30 天销量 ≥ 100 件",
  "confidence": 0.85,
  "ambiguous": true,
  "ambiguity_pattern_id": "1.1",
  "ambiguity_note": "近 30 天是滚动还是自然月待澄清",
  "default_assumption": "按滚动 30 天处理",
  "clarification_question": "本规则中'近 30 天'是按活动开始日往前推 30 个自然日（滚动窗口），还是上一个完整自然月？"
}
```

Agent 同时在 `clarification_questions` 列表中输出所有澄清问句，等待 `human_review_gate` 收集人工答复。
