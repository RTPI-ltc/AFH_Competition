# 天猫超级品牌日（Tmall Super Brand Day）规则知识库

> **聚焦赛题方向 C**：本文档只整理 Agent 在解析活动规则、生成检查清单、提示风险点时需要的"可被机器核查的规则条款"。不包含战略向（流量包/PR/数据看板）与合同向（佣金/保证金/扣点）内容——那些属于赛题方向 A/D。
> 适用场景：周大福作为珠宝品类品牌方在天猫报名"超级品牌日"。规则以平台公开运营手册、商家中心规则为参考整理，落地以当期商家后台报名页为准。

---

## 一、商家侧：可被解析为检查项的规则条款

### 1.1 资质门槛（资质核验类）

| 检查项 | 门槛 | 备注 |
|--------|------|------|
| 经营主体 | 必须为天猫旗舰店（官方旗舰店 / 专卖店） | 淘宝 C 店、企业店不可报名 |
| 行业资质 | 商标权属证明、贵金属生产/销售许可（黄金珠宝类目） | 一口价饰品需 CMA 抽检报告样张 |
| 店铺评分 | DSR ≥ 4.7 | 取报名当日值 |
| 违规记录 | 近 180 天无 "严重违规扣 12 分" | 命中即不可报名 |
| 经营时长 | 旗舰店开店 ≥ 12 个月 | 新店走"新锐品牌日" |

### 1.2 SKU 选品条件（数量类 + 数据类 + 参数精度类）

| 检查项 | 门槛 |
|--------|------|
| 报名 SKU 数 | ≥ 200 款 |
| 超级单品（爆款）数 | ≥ 5 款 |
| 品类覆盖 | 覆盖品牌主营 ≥ 3 个二级品类 |
| 单 SKU 近 90 天店内销量 | ≥ 50 件 或 近 90 天搜索词曝光 ≥ 1 万 |
| 单 SKU 近 30 天好评率 | ≥ 95% |
| 单 SKU 近 30 天差评率 | ≤ 1% |
| 新品占比（近 90 天上新） | ≥ 30% |
| 单 SKU 报名库存承诺 | ≥ 500 件 或 ≥ 日均销量 × 10 天 |

### 1.3 参数精度类（针对珠宝类目）

| 品类 | 必须填写的参数及精度 |
|------|----------------------|
| 黄金类 | 克重精度到 0.01g；成色（足金 999 / 999.9 / 999.99） |
| 镶嵌类 | 主石克拉数 0.01ct；颜色（D-Z）；净度（FL-I3）；切工（EX/VG/G/F/P） |
| 一口价类 | 明确标"一口价 / 计重金"二选一，禁止混标 |

### 1.4 价格管控（核心，全部高置信度数值规则）

> Agent 解析时全部标记为 `confidence ≥ 0.9`，并把"近 30 天"按**滚动 30 天**处理。

| 检查项 | 规则 |
|--------|------|
| 活动价 | ≤ 近 30 天店内最低成交价（含优惠后实付） |
| 活动价 vs 全网 | ≤ 全网同款最低价（京东、抖音、得物、品牌官网等含） |
| 大促额外让利 | 部分期次要求活动价 ≤ 历史 365 天最低 × 95% |
| 划线价合规 | 必须是近 30 天内有实际成交、未划线的店铺销售价（不可用厂商指导价 / 吊牌价） |
| 价保 | 价保 15 天，按"到手价"口径退差 |
| 满赠估值 | 赠品估值 ≤ 商品本身 30%；超过视为变相打折，回溯计入近 30 天最低价 |

### 1.5 品类互斥规则

| 互斥项 | 说明 |
|--------|------|
| 跨店满减 | 默认参加官方跨店满减（如 "每满 300 减 50"），不可单独退出 |
| 品牌补贴 | 与"百亿补贴"、"聚划算品牌闪购"二选一作为前台主入口 |
| 同期活动 | 不可同期参加"行业品类日"、"超级品类日"（大促期间另议） |
| 跨子品牌店铺 | 周大福旗舰店、周大福 LOLA、SOINLOVE、T MARK 等需各自单独报名 |

## 二、消费者侧（仅商家需在客服话术中告知的部分）

> 完整消费者规则见 `consumer_benefits.md`，本节仅列与"商家是否能做某动作"直接相关的合规条款。

| 商家必须执行 | 说明 |
|-------------|------|
| 价保 15 天自动退差 | 客服话术统一答复"自支付完成起 15 天，按到手价口径自动退差" |
| 满赠不超 30% | 商家自营满赠不可让赠品价值超过 30% |
| 跨店满减摊回逻辑 | 部分退货后剩余商品按比例摊回满减，客服需向消费者告知 |
| 价保不含会员价 | "新客券、定向券、88VIP 95 折"形成的低价不在价保范围 |

## 三、商家关注的流程与环节（赛题视角 → 检查清单骨架）

> Agent 在 `checklist_builder` 节点应以此结构为骨架。详见 `merchant_process_summary.md`。

| 阶段 | 关键检查项 | Agent 输出动作 |
|------|-----------|----------------|
| T-30 资质核验 | 营业执照、商标证、贵金属许可证在有效期；DSR ≥ 4.7；近 180 天无 A 级违规 | 缺/临期 → `high` |
| T-30 价格盘点 | 拉取报名 SKU 的近 30 天 + 近 365 天最低价、全网同款价 | 穿底 → `high` |
| T-30 互斥盘点 | 同 SKU 是否参与了其他互斥活动（聚划算、品牌日等） | 命中 → `high` |
| T-15 报名提报 | 上传 SKU 列表、价格清单、库存承诺、商品证书电子件 | 缺件 → `medium` |
| T-7 预热 | 私域触达、直播预热、店铺装修打"超品日"角标 | — |
| T0 当日盯盘 | 价格 / 券 / 跨店满减是否生效；流量 / 库存 / 转化盯盘 | 异常 → `high` |
| T+15 价保 | 每日跑批退差，避免售后险扣分 | — |

## 四、常见踩坑（Agent 高优先级风险点输出参考）

| 风险点 | 触发场景 | 风险等级 | Agent 建议 |
|--------|---------|---------|------------|
| 历史最低价穿底 | 漏查近 30 天某次秒杀价 | high | 拉订单库核查 |
| 划线价虚高 | 用厂商指导价 / 吊牌价 | high | 改用近 30 天实际成交价 |
| 赠品估值超 30% | 满赠中大额赠品 | medium | 拆为"满赠 + 直降"双结构 |
| 互斥活动冲突 | 同期已报聚划算 | high | 二选一 |
| 库存承诺不达 | 实际库存 < 报名承诺 | high | 同步仓库或下架该 SKU |
| DSR 跌破阈值 | DSR < 4.7 | high | 先暂停售后高风险订单 |
| 新品占比不足 | 近 90 天上新 < 30% | medium | 从上新中补充 |
| 参数精度不全 | 钻石 SKU 缺净度等级 | medium | 补全后再提交 |
