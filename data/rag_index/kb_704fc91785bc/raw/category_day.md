# 行业品类日 / 超级品类日规则知识库

> **聚焦赛题方向 C**：品类日 / 类目日是**赛题示例规则"已参加品牌日不可重复报名"互斥规则的对手方**，Agent 必须能识别其与品牌日的关系。

---

## 一、品类日 vs 品牌日 的核心差异

| 维度 | 品牌日（Super Brand Day） | 品类日（Category Day） |
|------|-----------------------------|--------------------------|
| 主角 | 单一品牌 | 单一类目（如"金饰类目日"、"钻饰类目日"） |
| 参与商家数 | 1 个品牌独占 | 多个品牌共同参与 |
| 流量入口 | 品牌专属焦点图 | 类目专属频道 |
| 价格门槛 | 品牌让利 ≥ 15% | 类目内 SKU 让利 ≥ 10% |
| SKU 数量 | ≥ 200 款 | 单品牌 ≥ 30 款，类目总 ≥ 500 款 |
| 互斥关系 | 与品类日**互斥**（同期不可同 SKU 参加） | 与品牌日**互斥** |

> ⚠️ **赛题示例规则中"已参加品牌日的商品不可重复报名"**，Agent 解析时需识别：当前活动若是"品类日"，则需检测该 SKU 是否曾报名"品牌日"。

## 二、品类日的可解析规则条款

### 2.1 资质门槛

| 检查项 | 门槛 |
|--------|------|
| 经营主体 | 旗舰店 / 专卖店 |
| 类目覆盖 | 必须经营对应类目（如金饰类目日仅黄金类商家） |
| DSR | ≥ 4.6（比品牌日略低） |
| 违规记录 | 近 90 天无 A 级违规 |

### 2.2 SKU 选品条件

| 检查项 | 门槛 |
|--------|------|
| 单品牌报名 SKU 数 | ≥ 30 款 |
| 类目内总 SKU 数（全平台） | ≥ 500 款 |
| 单 SKU 近 90 天销量 | ≥ 30 件 |
| 单 SKU 好评率 | ≥ 95% |
| 库存承诺 | ≥ 200 件 或 日均销量 × 7 天 |

### 2.3 价格管控

| 检查项 | 规则 |
|--------|------|
| 活动价 ≤ 近 30 天最低价 | 与超品日一致 |
| 让利幅度 | ≥ 10%（比超品日的 15% 略松） |
| 划线价合规 | 近 30 天内有实际成交 |
| 价保 | 7 天（比超品日 15 天短） |

### 2.4 品类日特有的互斥规则

| 互斥项 | 说明 |
|--------|------|
| **与品牌日互斥** | 同 SKU 不可在同期同时参加品牌日和品类日 |
| 与百亿补贴 | 可叠加（品类日不算"流量频道"，可以共存） |
| 与聚划算品牌闪购 | 可叠加 |
| 跨店满减 | 默认参加 |

## 四、互斥规则识别样例（关键，对应赛题）

### 4.1 互斥检测算法

```python
def is_mutually_exclusive(sku, target_campaign):
    """
    target_campaign: 当前要报名的活动
    sku.active_campaigns: SKU 当前在售的活动列表
    """
    # 取出 SKU 已参加的所有活动
    active = sku.active_campaigns

    # 互斥规则矩阵（详见 ../rule_library/mutual_exclusion_matrix.md）
    exclusive_pairs = [
        ("brand_day", "category_day"),  # 品牌日 vs 品类日
        ("super_brand_day", "super_category_day"),  # 超品日 vs 超级品类日
        ("juhuasuan", "baiyibutie"),  # 聚划算 vs 百亿补贴
    ]

    for camp in active:
        for a, b in exclusive_pairs:
            if (camp == a and target_campaign == b) or \
               (camp == b and target_campaign == a):
                return True, f"已参加 {camp}, 与 {target_campaign} 互斥"
    return False, None
```

### 4.2 赛题示例规则的实际识别

```text
[规则] "已参加品牌日活动的商品不可重复报名"
[当前活动] 天猫 618（含品类日子频道）

[Agent 检测]
- 拉取所有报名 618 的 SKU
- 对每个 SKU 检测 sku.active_campaigns 是否包含 "brand_day" / "super_brand_day"
- 命中 → high 风险，建议下架该 SKU
```

## 五、常见踩坑

| 风险点 | 触发场景 | 风险等级 |
|--------|---------|---------|
| 同 SKU 双报品牌日 + 品类日 | 互斥规则未识别 | high |
| 类目划分错配 | 误把镶嵌类 SKU 报金饰类目日 | medium |
| 让利幅度按品牌日 15% 填品类日 10% | 规则混淆 | low（多让利不算违规但浪费预算） |
| 价保 15 天 vs 7 天混淆 | 商家按 15 天承诺消费者，平台只赔 7 天 | medium |
