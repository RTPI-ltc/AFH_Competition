# 内部营销项目规则知识库（赛题原文提及）

> **赛题原文："企业 X 在参与各类平台活动**或内部部署营销项目**时..."**
> 内部营销项目是周大福自有的、不依赖电商平台规则的营销活动。规则口径与电商平台差异显著，Agent 必须能识别并分别处理。

---

## 一、内部营销项目的典型类型

| 项目类型 | 周期 | 触发场景 |
|---------|------|---------|
| **会员日 / 黄金会员日** | 每月固定日 | VIP 会员复购激励 |
| **店庆 / 周年庆** | 每年 9 月（周大福创立纪念） | 全集团统一促销 |
| **婚嫁季营销** | 5–6 月、9–10 月 | 婚嫁高峰针对性营销 |
| **节日营销** | 春节、情人节、520、七夕、母亲节、圣诞 | 节日礼赠场景 |
| **以旧换新季** | 每季度 1 次 | 旧金回收 + 新款销售 |
| **传承系列发布** | 不定期 | 古法金新品发布 |
| **清仓 / 库存优化** | 季末 / 年末 | 库存周转 |

---

## 二、内部营销 vs 电商平台规则 的核心差异

| 维度 | 电商平台规则 | 内部营销项目 |
|------|--------------|--------------|
| 规则来源 | 平台公开 / 商家中心 | 集团总部 + 区域负责人下发 |
| 价格管控 | "近 X 天最低价" 硬约束 | **自定**（可豁免近 X 天最低价） |
| 互斥关系 | 平台间互斥矩阵 | **与平台活动需手动互斥**（重要） |
| SKU 选品 | 平台数据门槛筛选 | 区域 / 渠道差异化选品 |
| 让利预算 | 商家自承担 + 平台补贴 | 集团内部预算 |
| 渠道 | 单一电商平台 | **同时跨线上 + 线下门店** |
| 审批 | 平台小二 | 集团内部审批（详见 `../chow_tai_fook/human_review_routing.md`） |

> ⚠️ **关键风险**：内部营销项目的"特价款"如果同时挂在天猫店铺，会被天猫风控算法识别为"穿底"，导致店铺评分下降。Agent 必须能识别这种"线下+线上 联动"风险。

## 三、可解析的内部营销规则模板

### 3.1 会员日

| 检查项 | 典型规则 |
|--------|---------|
| 适用对象 | VIP1+ 会员（年消费 ≥ 5000） / VIP3+ 会员（年消费 ≥ 5 万） |
| 优惠形式 | 会员专享价 / 满减券 / 会员积分翻倍 |
| 价格 | 会员价 ≤ 普通价 95 折（部分品类除外） |
| 互斥 | 不与天猫超品日叠加 |
| 渠道 | 仅会员专属页面 / 微信小程序 |
| 时间 | 当月第 X 天 0:00 ～ 24:00 |

### 3.2 店庆

| 检查项 | 典型规则 |
|--------|---------|
| 全集团统一参与 | 是 |
| 让利幅度 | 全场 8 折起，部分品类 7 折 |
| 互斥 | 与同期电商品牌日、品类日**互斥**（关键风险点） |
| 渠道 | 线上 + 全国线下门店 |
| 时间 | 9 月某 7–14 天 |

### 3.3 婚嫁季

| 检查项 | 典型规则 |
|--------|---------|
| 适用品类 | 婚嫁三金（项链 / 手镯 / 戒指）+ 求婚钻戒 + 对戒 |
| 优惠形式 | 婚嫁套餐价（三件成套优惠） |
| 价格 | 套餐价较单买总价让利 ≥ 10% |
| 渠道 | 线上 + 线下门店均可 |
| 互斥 | 与天猫钻饰类目日**可叠加**（属同类目宣传） |

### 3.4 以旧换新

| 检查项 | 典型规则 |
|--------|---------|
| 适用对象 | 持有周大福旧饰品的消费者 |
| 旧金回收 | 按当日金价 × 95% 回收（扣 5% 折损） |
| 新金价 | 新款 = 旧金克数 × 当日金价 + 工费 |
| 手续费 | 工费 30%（活动期间可减半） |
| 互斥 | 以旧换新差额部分**不享受**电商平台活动券 |
| 渠道 | 仅线下门店 / 部分线上预约换货 |

### 3.5 清仓 / 库存优化

| 检查项 | 典型规则 |
|--------|---------|
| 适用 SKU | 季末 / 年末高库存 SKU、过季款、停产款 |
| 让利幅度 | 5–6 折（远超平台规则） |
| 互斥 | **与所有电商品牌日 / 品类日互斥**（清仓价会穿天猫近 30 天最低） |
| 渠道 | 仅奥莱店铺 / 特定清仓页面 |
| 库存承诺 | 售完即止，不补货 |

## 四、Agent 解析内部营销规则的特殊注意

### 4.1 渠道隔离

```text
[规则] "店庆全场 8 折"
[Agent 解析]
- 渠道：线上（旗舰店 + 奥莱店）+ 线下全国门店
- 若线上店铺同期参与天猫品牌日 → 高风险（店庆价穿天猫近 30 天最低）
- 主动澄清："本店庆是否包括天猫官方旗舰店？"
```

### 4.2 价格穿底风险（最高频）

```text
[场景] 内部清仓季 6 折，天猫旗舰店同时挂该 SKU
[Agent 检测]
- 拉取该 SKU 在天猫的"近 30 天最低成交价"
- 若清仓 6 折价 < 天猫近 30 天最低 → high 风险
  → 触发天猫价格管控算法
  → 该店铺所有 SKU 可能被降权
- 建议：清仓 SKU 同时在天猫下架，或线上线下分 SKU 编码运营
```

### 4.3 以旧换新差价与平台券叠加

```text
[规则] "以旧换新差价部分不享受电商平台券"
[Agent 解析]
- 商详页必须明示
- 客服话术统一："以旧换新差价不可使用平台券，仅可使用店铺以旧换新专享券"
- 风险：消费者误叠加平台券 → 客诉
```

## 五、常见踩坑

| 风险点 | 触发场景 | 风险等级 |
|--------|---------|---------|
| 清仓价穿天猫近 30 天最低 | 同 SKU 跨渠道 | high |
| 店庆与天猫品牌日同期 | 互斥未识别 | high |
| 以旧换新差价叠加平台券 | 商详页未明示 | medium |
| 内部营销价被外渠抓取 | 影响百亿补贴"全网最低"承诺 | high |
| 线上线下 SKU 同编码 | 库存承诺与实际不符 | medium |
